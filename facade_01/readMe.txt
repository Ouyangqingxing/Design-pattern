设计模式： 外观/门面 模式。
定        义： 为子系统中的一组接口提供一个一致的界面，此模式定义了一个高层接口，这个接口使得这一子系统更加容易使用。		 
解决问题： 无。本实例信息中心为外观类，向外界提供对子系统（这里就是学生教师课程类）的访问。此模式在平时开发中容易想到。

Design pattern: Facade pattern.
Definition:Provide a unified interface to a set of interfaces in a subsystem, this model defines a higher-level interface, this interface makes the system easier to use.
Solve the problem:Nothing.The center of information is the facade class and it provide the chance to outside to visit the sub system.This pattern is easy to getting.
